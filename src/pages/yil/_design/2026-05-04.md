## Overview

This page ports BMW M's engineered black-canvas visual language onto a day of systems thinking about agent skill architecture, class-name audit discipline, and content-driven design. The pairing is precise: BMW M's aesthetic IS a routing system — every visual decision has a rule, a cost, and a placement, exactly like a well-written skill specification. The M tricolor stripe, used sparingly as a brand-register mark in the original, becomes the page's only decorative element here, echoing how the day's core insight uses constraint to create signal.

Surface rhythm follows BMW M: black hero → near-black quote band → elevated surface-card (skills cards) → black (Tailwind code window) → surface-soft (design systems split) → near-black (meta full-bleed callout) → black references + footer. All headlines are Inter 700 in UPPERCASE; body is Inter 300 — the weight contrast is the editorial signature BMW M's system is built on. Border radius is 0px throughout; the only circular elements would be icon buttons if any appear.

## Do's and Don'ts

### Do's
- Keep all section labels in UPPERCASE with ~1.5px letter-spacing — they are "machined" labels, not casual category tags
- Maintain the 700 headline / 300 body weight contrast at all times — never use 400 or 500 for either display or body
- Treat the M tricolor stripe as a sparingly-used signature mark; it appears at the hero header and as a single-color echo at the meta callout band — nowhere else

### Don'ts
- Don't add rounded corners to cards, code windows, or button-like elements — 0px radius is the discipline; it reads as engineered, not consumer-tech
- Don't use the M tricolor colors (#0066b1, #1c69d4, #e22718) as fill backgrounds — they appear only as text accents, border accents, and the 4px stripe divider
- Don't use a heavier body weight for inline emphasis — use a small UPPERCASE inline badge or a bright-white label instead; the 700/300 weight contrast IS the page's editorial voice
